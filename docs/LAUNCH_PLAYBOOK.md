# Launch Playbook

## Positioning

Do not market Aftercare as "another AI coding agent".
Market it as the **post-development governance layer for coding agents**.

## Best launch message

"Your agent built the feature. Aftercare makes sure it did not quietly damage the repo."

## What gets stars in this category

1. Sharp problem statement
2. Host compatibility
3. Screenshots or examples of useful output
4. Small install surface
5. Distinct opinions
6. Evidence that it improves real workflows

## Recommended launch package

- polished README
- one diagram showing the post-dev workflow
- three example reports from realistic diffs
- host install guides
- a short comparison section explaining why this is not just generic code review

## First community-facing roadmap

- add more scenario guides
- add more example reports
- add optional marketplace distribution
- add benchmark corpus
- add optional CI integration docs
