## Design rule

Every skill should do one of three things well:

1. detect a class of post-dev problems
2. turn findings into a safe action plan
3. control the decision to continue, merge, or stop

## Output consistency

Review skills should converge on the same report structure:

1. summary
2. findings by severity
3. why it matters
4. recommended next action
5. merge or continue decision if applicable
